# BetterTeam | 更好的队伍

> **Mod ID:** `better_team`

---

## 简介

**BetterTeam** 是一个适用于 **Fabric 1.21.10** 的服务端模组，用于对原版 `/team` 队伍系统进行小幅增强。

在原版中，只有同队玩家之间默认无法互相攻击，但队伍内的其他实体仍然会受到同队玩家的伤害（例如：红队玩家依然可以攻击加入红队的僵尸）。

安装 BetterTeam 后，这一规则将扩展到**所有**加入同一原版队伍的实体：
* 同队实体之间无法互相造成伤害。
* 同队来源的无差别攻击也会被正确拦截。
* 同队发射的弹射物不会被同队实体阻挡，而是像原版同队玩家判定一样直接穿透。

*注：本模组仅作用于原版 `/team` 队伍系统，不提供新的队伍系统，也不包含额外配置项。*

---

## 功能特性

### 1. 扩展原版“同队免伤”判定到所有实体
原版的同队免伤主要体现在玩家之间。BetterTeam 会将这套逻辑扩展到所有加入同一队伍的实体，使它们无法受到来自同队实体的伤害。这意味着以下情况都会被阻止：
* 同队玩家攻击同队生物
* 同队生物攻击同队玩家
* 同队生物攻击同队生物

### 2. 处理多种伤害来源
BetterTeam 不仅处理普通近战，还会处理来自同队实体的其他伤害来源，包括但不限于：
* 近战攻击
* 弓箭、弩箭、三叉戟、火球等投射物
* 河豚、闪电、唤魔者尖牙
* TNT、末影水晶爆炸
* 药水、滞留药水、区域效果云
* 驯服生物替主人造成的伤害

### 3. 同队弹射物可穿透同队实体
同队发射的弹射物不会被同队实体阻挡。这可以让同队协作更加自然，也更接近原版同队玩家之间的判定体验。
* **效果**：红队玩家射出的箭会**穿过**红队实体，并且该箭仍可正常命中后方的**非红队**实体。

### 4. 不影响环境伤害
BetterTeam 只处理来自同队实体的伤害。环境伤害不受影响，例如：
* 掉落伤害
* 火焰或岩浆伤害
* 窒息伤害
* 虚空伤害
* 其他非同队实体直接造成的环境类伤害

## 安装与使用

### 安装方法
1. 安装对应版本的 Fabric Loader。
2. 将 BetterTeam 模组文件放入服务端的 `mods` 文件夹。
3. 启动服务器。

### 使用方式
本模组**不添加新命令**，直接增强原版 `/team` 系统。当玩家、生物或其他实体加入同一个原版队伍后，BetterTeam 会自动生效。

你仍然可以使用原版命令来管理队伍，例如：
```mcfunction
/team add red
/team join red PlayerName
/team join red @e[type=zombie,limit=1,sort=nearest]
```

## 效果示例

**示例 1：同队玩家无法攻击同队僵尸**
> * **假设**：玩家 Alex 在红队，一只僵尸也被加入红队。
> * **原版中**：Alex 仍然可以攻击这只红队僵尸。
> * **安装后**：Alex **无法**对这只红队僵尸造成伤害。

**示例 2：同队箭矢穿透队友**
> * **假设**：红队玩家向前方射箭，箭道中间站着红队实体，更远处站着非红队目标。
> * **安装后**：箭会**穿过**红队实体，并仍可命中后方的非红队目标。

**示例 3：同队来源的范围或特殊伤害无效**
> * **假设**：同队实体触发 TNT、药水、区域效果云、唤魔者尖牙或其他同队来源的伤害。
> * **安装后**：同队目标**不会**因此受到伤害。

## 适用场景

BetterTeam 适合用于以下类型的服务器，尤其适合需要让“玩家、宠物、召唤物、怪物单位”统一纳入原版队伍判定的场景：
* 小游戏服务器
* PVE / RPG 服务器
* 阵营对抗服务器
* 需要让召唤物、宠物或自定义队伍实体真正共享队伍保护的玩法环境

## 其他说明

### 设计目标
BetterTeam 的目标不是重写原版队伍系统，而是在尽可能保持原版使用方式的前提下，补全一个更一致的规则：**只要属于同一原版队伍，就不应互相造成伤害。**

### 兼容性说明
* 本模组仅针对原版 `/team` 队伍系统。
* 不包含对其他模组自定义队伍系统的适配说明。
* 无配置文件，无黑白名单，无额外命令。

### 已知限制
* 本模组的判定基础是原版 `/team`，未加入队伍的实体不会受到保护。
* 仅处理同队伤害相关逻辑，不改变其他原版战斗规则。
